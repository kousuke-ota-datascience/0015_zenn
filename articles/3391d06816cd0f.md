# 0. Introduction
## 0.1 目的
sys.path.append をやめ、src layout の自作 package を uv 管理の仮想環境に editable install する。

## 0.2 前提

- OS: Ubuntu 24.04
- Python 3.13
- PROJECT_ROOT 構成: 本記事最下部参照

# 1. uv のインストール

uv のインストール

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

PATH の設定

```bash
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
which uv
```

version の確認

```bash
uv --version
uv python list --only-installed
```

# 2. プロジェクト初期化

uv init にて、初期化を行う。

```bash
cd [PROJECT_ROOT]
uv init --python 3.13
cat .python-version
```

そうすると、主に以下のファイルが生成される。

- pyproject.toml: プロジェクト定義
- .python-version: このプロジェクトで使用する Python version
- main.py: uv init により作成されるサンプル entry point
- README.md: プロジェクト説明用ファイル

# 3. 自作 package を import 可能にする設定

## 3.1. pyproject.toml の設定値

pyproject.toml は以下の通りとなる

```toml
[project]
name = "zenn-workspace"
version = "0.1.0"
description = "Add your description here"
readme = "README.md"
requires-python = ">=3.13"
dependencies = [
    "causal-learn>=0.1.4.5",
    "dask>=2026.3.0",
    "econml>=0.16.0",
    "pandas>=3.0.2",
    "pyarrow>=24.0.0",
    "pyyaml>=6.0.3",
    "rdata>=1.0.0",
]

[tool.uv.workspace]
members = []

[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"

[tool.hatch.build.targets.wheel]
packages = ["shared/py/myproj/src/myproj"]

[dependency-groups]
dev = [
    "ipykernel>=7.2.0",
    "pytest>=9.0.3"
]
```

## 3.2. 重要な設定項目の意味

### 3.2.1. build-system

build-system を定義する。これがこの記事でやりたいことの本質となる

```toml
[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"
```

1. uv がこのリポジトリを Python package として扱う
1. uv sync / uv run で editable install される
1. site-packages の .pth 経由で shared/py/myproj/src が import path に入る
1. sys.path.append が不要になる

### 3.2.2. hatchling

- hatchling は、このプロジェクトを wheel として build するための build backend である。
- uv 自体は package の build backend ではないため、`[build-system]` で hatchling を指定する。
- 今回の目的は PyPI 配布ではなく、ローカル開発環境に editable install することである。

### 3.2.3. packages

- 以下のように記載することで、作成したpackageをimportできるようにする

```
packages = ["shared/py/myproj/src/myproj"]
```

- 注意点
    - distribution name は `[project].name` の `zenn-workspace` だが、Python コード上で import する名前は package directory 名の `myproj` である。
    - 今回の場合、`shared/py/myproj/src/myproj/__init__.py` が存在するため、`import myproj` が可能になる。


# 4. 依存関係の同期

## 4.1. uv add による必要パッケージのインストール

```bash
uv add causal-learn dask econml pandas pyarrow pyyaml rdata
uv add --dev pytest ipykernel
```

## 4.2. uv sync による設定の同期

- uv sync は `pyproject.toml` と `uv.lock` に基づいて `.venv` を作成・更新する。
- `uv.lock` がなければ依存解決を行い、解決結果を `uv.lock` に記録する。

```
uv sync
```

## 4.3. uv.lock について

- uv sync の実行後は、以下2つのプロジェクト定義情報が作成されている状態となる
    - pyproject.toml : 直接依存の宣言
    - uv.lock は解決済み依存の固定。
- チームでは pyproject.toml と uv.lock を両方 commit する。
- 他メンバは uv sync で同じ依存環境を再現する。


# 5. 動作確認

## 5.1. python 実行時 の import 確認

以下のコマンドを実行することで、myproj packageがimportできているかを確認する

実行コマンド
```bash
uv run python -c "import myproj; print(myproj.__file__)"
```

期待結果
```
[PROJECT_ROOT]/shared/py/myproj/src/myproj/__init__.py
```

## 5.2. notebook kernel の確認

```bash
uv run python -m ipykernel install --user --name zenn-workspace --display-name "Python (zenn-workspace)"
```

期待結果
```
Installed kernelspec zenn-workspace in /home/[USER NAME]/.local/share/jupyter/kernels/zenn-workspace
```

## 5.3. .pth の確認

site-packages の .pth 経由で shared/py/myproj/src が import path に入っているか確認

実行コマンド
```bash
find .venv/lib -path '*site-packages/*.pth' -print -exec cat {} \;
```

期待結果
```bash
[PROJECT_ROOT]/shared/py/myproj/src
```
