    - VS Code から Codex につなげようとすると時間がかかり過ぎで表示されない

- 原因
    - ローカルのvscodeの設定がなんらかおかしくなっていた
    - （確定ではない） の設定

# 1. 事象
- VS Code 上で、 OUTPUT(terminal を開くとその左にあるタブから開ける)を開き、codexの出力を確認すると以下の出力

```
> [error] [CodexMcpConnection] cli: message="codex_app_server: Codex could not find bubblewrap on PATH. Install bubblewrap with your OS package manager. See the sandbox prerequisites: https://developers.openai.com/codex/concepts/sandboxing#prerequisites. Codex will use the bundled bubblewrap in the meantime."
```

そこで、bubblewrapのインストール
```
sudo apt update
sudo apt install -y bubblewrap
```

これまでBUBBLEWRAPとやらがなくても動いていたのに、なぜ？

可能性が高いのは、Codex CLI / VS Code Codex拡張が更新され、Linux sandboxの既定実装・起動時チェックが bubblewrap 前提に寄ったため
必要になった、というよりも警告が見えるようになったというのが正しい

その後、このようなエラー

```
[error] [CodexMcpConnection] cli: message="codex_app_server: Codex's Linux sandbox uses bubblewrap and needs access to create user namespaces."
```

以下のコマンドで確認する

```bash
$ which bwrap
/usr/bin/bwrap

$ bwrap --version
bubblewrap 0.9.0

$ sysctl kernel.unprivileged_userns_clone 2>/dev/null || true
kernel.unprivileged_userns_clone = 1

$ sysctl user.max_user_namespaces 2>/dev/null || true
user.max_user_namespaces = 31440

$ sysctl kernel.apparmor_restrict_unprivileged_userns 2>/dev/null || true
kernel.apparmor_restrict_unprivileged_userns = 1

$ aa-status 2>/dev/null | head -40 || true
apparmor module is loaded.

$ cat /etc/os-release
PRETTY_NAME="Ubuntu 24.04.4 LTS"
NAME="Ubuntu"
VERSION_ID="24.04"
VERSION="24.04.4 LTS (Noble Numbat)"
VERSION_CODENAME=noble
ID=ubuntu
ID_LIKE=debian
HOME_URL="https://www.ubuntu.com/"
SUPPORT_URL="https://help.ubuntu.com/"
BUG_REPORT_URL="https://bugs.launchpad.net/ubuntu/"
PRIVACY_POLICY_URL="https://www.ubuntu.com/legal/terms-and-policies/privacy-policy"
UBUNTU_CODENAME=noble
LOGO=ubuntu-logo
$ 
```


確認結果から、Ubuntu 24.04のAppArmorによる unprivileged user namespace 制限によるもの
開発用マシンなので、APPARMORの制限を外す

```
echo 'kernel.apparmor_restrict_unprivileged_userns=0' | sudo tee /etc/sysctl.d/99-codex-apparmor-userns.conf
sudo sysctl --system
```
